Why

You may be wondering - why we made this project?

  • Our upstream project Void Linux has "Linux" as it's kernel by default, which has some closed source blobs that can't be read, studied, modified or learned from. It stops you from enjoying total software freedom.
  • Void Linux does not allow custom kernel builds on their project, so liberated kernels can't be allowed. (It's not their fault. They have limited resources.)
  • Many packages on Void Linux repos are not fully libre or has major issues that harms user freedom. e.g. Abiword has hard-coded nonfree fonts, p7zip has nonfree unrar. Full list here
  • Many packages suggest or forcefully push closed source, privacy disrespecting services, features or optional dependencies to you. e.g. Firefox, Chromium has telemetry enabled by default.

We can't directly contribute these fixes to Void Linux. So we had no other way but to create this separate project. We also had to "blacklist" some packages that can never be "liberated", so having a separate project makes sense.

Solution

This project brings Linux-libre kernel on Void Linux ecosystem. Linux-libre is a fork of the original Linux kernel that removes closed source proprietary blobs that are inherently present on a typical Linux kernel. So now you can enjoy freedom even on kernel level.

The project also provides a cleaner, "liberated" version of semi-free packages to ensure user freedom. (NOTE: All the packages are not "liberated" yet. We're working on it.) We also have a "your-freedom" package to keep out nonfree packages from being installed on your system by accident.

The goal is to make your system clean and free from proprietary or nonfree stuff.

Install

It's fairly easy to install. Just install Void Linux first, then run:

$ wget https://github.com/drake-newell/void-packages/raw/master/void-liberation || curl -LO https://github.com/drake-newell/void-packages/raw/master/void-liberation
$ chmod +x void-liberation
$ sudo ./void-liberation

If you want to install from source, follow our guide on project README.
